============================================================
 OE SERVICES WEBSITE — FOR THE DEVELOPER
============================================================

Hi — this folder is a complete, working website (HTML/CSS/JavaScript,
no build step, no frameworks). You can open index.html in a browser and
it runs as-is. It's meant as the build spec / starting point for the
live site.

WHAT TO READ FIRST
------------------
Open  README.md  — it explains the file structure, how the estimator
pricing works, and (importantly) the two things that must be finished
before launch:

  1) PROOF CONTENT — the home page has placeholders marked [REPLACE: ...]
     (reviews, ratings, project count, years in business). They're
     outlined in yellow on screen. The business owner will supply the
     real content; swap it in and remove every [REPLACE] marker. Do NOT
     invent numbers or reviews.

  2) LEAD SUBMISSION — the estimate form currently shows a confirmation
     screen but does not send the lead anywhere yet. There's a clearly
     marked hook in assets/estimator-ui.js (search "submission hook").
     Wire it to the owner's email / CRM / form service.

QUICK START
-----------
  - Run locally:   python3 -m http.server 8080   →   http://localhost:8080
  - Deploy:        drag the folder into Vercel/Netlify, or upload to any
                   static host. No server required.

  The site already includes structured data (JSON-LD), sitemap, AI-crawler
